2. ФУНКЦИОНАЛ ПОЛЬЗОВАТЕЛЯ (USER CAPABILITIES)
Пользователь имеет доступ к трем уровням взаимодействия через интерфейс Streamlit.
A. МУЛЬТИМОДАЛЬНЫЙ ВВОД (INPUT) 181818181818181818


Пользователь может загружать данные в любом виде ("Всеядный движок"):
1. Документы: PDF, DOCX, TXT, RTF (Резюме, Описания вакансий).
2. Изображения: JPG, PNG (Скриншоты профилей LinkedIn, фото документов).
3. Аудио: MP3, WAV, M4A, OGG (Голосовые заметки рекрутера, записи интервью).
B. РЕЖИМЫ РАБОТЫ (MODES)
1. JD Architect: Создание профиля вакансии из голосового описания или текста. Настройка контекста (War/Peace Time).
2. Forensic Assessment: Загрузка резюме -> Получение отчета "Рентген".
3. Battle Matrix: Загрузка 2+ кандидатов -> Генерация сравнительной таблицы.
4. Voice Briefing: Генерация аудио-отчета (TTS), чтобы слушать вердикт в дороге.
C. СТРУКТУРА ОТЧЕТОВ (OUTPUT STRUCTURE)
Каждый отчет AI генерируется в формате Markdown по следующей структуре:
1. EXECUTIVE VERDICT:
   * Fit Score: Оценка соответствия (0-100%).
   * Decision: Go / No-Go / Conditional.
   * Context: Подходит ли для "War Time" или "Peace Time".
2. TRUTH AUDIT (ДЕТЕКТОР ЛЖИ):
   * Выявленные пробелы в датах.
   * Использование пассивного залога ("Мы" вместо "Я").
   * Отсутствие цифр/фактов.
3. STRATEGIC PROFILE:
   * Jaques Level: Текущий уровень (например, Level 4 - Strategic).
   * Hogan Risks: Красные флаги (например, "High Risk: Skeptical").
   * Learning Agility: Оценка обучаемости.
4. INTERVIEW GUIDE:
   * 3-5 вопросов для проверки выявленных рисков ("Drill Down").

3. ТЕХНИЧЕСКИЙ СТЕК И УСТАНОВКА (SYSTEM REQUIREMENTS)
Для корректной работы платформы a.py необходимо следующее окружение.
A. СИСТЕМНЫЕ ТРЕБОВАНИЯ
* ОС: Windows, macOS или Linux.
* Python: Версия 3.9 или выше.
* FFmpeg: (Опционально) Для обработки аудиофайлов в Linux/Windows.
B. НЕОБХОДИМЫЕ БИБЛИОТЕКИ (DEPENDENCIES)
Установите через pip install:
1. streamlit
   * Назначение: Веб-интерфейс, управление сессиями, сайдбар, файловый загрузчик.
2. openai
   * Назначение: Доступ к API (GPT-4o для логики, Whisper для транскрипции аудио, TTS-1 для синтеза речи).
3. pandas
   * Назначение: Создание датафреймов для "Матрицы Решений" (сравнение кандидатов).
4. pypdf
   * Назначение: Извлечение текста из PDF-файлов (резюме).
5. python-docx
   * Назначение: Извлечение текста из файлов MS Word (.docx).
C. МОДЕЛИ И API
* LLM Core: GPT-4o (Рекомендуется для Multimodal) или Qwen 2.5/3 (Локально через Ollama).
* Audio-to-Text: Whisper-1.
* Text-to-Speech: TTS-1 (Voice: Onyx).
D. КОНФИГУРАЦИЯ (SETTINGS)
В коде a.py жестко заданы параметры для обеспечения Forensic-стандарта:
* TEMPERATURE = 0.0 (Исключение галлюцинаций).
* SYSTEM_PROMPT (Включает в себя сжатую методологию из Блоков 1-5).
