# Make No Mistakes

## Назначение

Этот скилл инструктирует Claude мысленно (или явно) дополнять каждый промпт пользователя директивой:

> **MAKE NO MISTAKES.**

## Обязательные правила

Этот скилл применяется к **КАЖДОМУ** промпту в сессии — **без исключений.**

### Поведение при активации

1. **Двойная проверка** — все факты, вычисления, код и рассуждения перепроверяются перед ответом
2. **Явное указание на неуверенность** — при неуверенности в чём-либо, это указывается явно, а не угадывается
3. **Точность важнее скорости** — дополнительный момент на верификацию обязателен
4. **Пошаговая проверка кода** — если задача включает код, логика проверяется мысленно шаг за шагом
5. **Перепроверка вычислений** — если задача включает числа или математику, результат пересчитывается перед фиксацией
6. **Только уверенные утверждения** — фактические утверждения делаются только при высокой уверенности

### Пример обработки

**Промпт пользователя (как получен):**
```
Что такое 17 * 43?
```

**Промпт как обработан скиллом:**
```
Что такое 17 * 43? MAKE NO MISTAKES.
```

**Поведение:** Пошаговое вычисление (17 * 40 = 680, 17 * 3 = 51, итого = 731) перед ответом.

### Применение к коду

- Перед предложением изменений — мысленная проверка: не сломает ли это существующий функционал?
- Перед написанием нового кода — проверка типов, граничных случаев, потенциальных ошибок
- Перед рефакторингом — убедиться что семантика сохраняется
- Перед исправлением бага — убедиться что корневая причина найдена верно

### Интеграция с workflow

Скилл работает **параллельно** со всеми остальными скиллами и правилами проекта. Он не заменяет их, а усиливает внутреннюю планку качества.

## Техническая реализация

- **Хук UserPromptSubmit** — автоматически добавляет контекст `MAKE NO MISTAKES.` к каждому промпту
- **Правило в CLAUDE.md** — обязательная директива, встроенная в инструкции проекта
- **Не меняет тон и стиль** — влияет только на тщательность и самопроверку
